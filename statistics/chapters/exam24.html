第24章「クラスター分析」のキーポイント

クラスター分析とは，似ているデータ同士をグループ（クラスター）にまとめる手法です． 教師なし学習の代表例で，あらかじめグループ数やラベルが分からないデータに使います．

手法概要特徴
最近隣法（単連結法） 2クラスター間の距離 ＝ 最も近い点同士の距離（min） 細長い鎖状のクラスターを作りやすい
最遠隣法（完全連結法） 2クラスター間の距離 ＝ 最も遠い点同士の距離（max） コンパクトな球状のクラスターを作りやすい
K-means 法 K 個の中心（セントロイド）を反復的に更新してクラスターを形成 非階層的．K を事前に指定する必要がある
デンドログラム（樹形図）の読み方

階層的クラスタリングの結果はデンドログラム（樹形図）で表示されます． 縦軸（Height）はクラスターが結合されたときの距離を表し，高い位置で結合されるほど「似ていない」ことを意味します．

問24.1 — 問題文（TripAdvisor レビューデータのクラスタリング）

問24.1 [1] — デンドログラムの対応とユーザ番号の特定

解答・解説
解法のポイント：「最初に結合される2点」から順番に特定する

階層的クラスタリングでは，距離行列の中で最も小さい値を持つ2点が最初に結合されます． この「最初に結合される組」は，最近隣法・最遠隣法のどちらでも同じです． そこから順に，各手法独自の距離更新ルールに従って結合を繰り返します．

Step 1：最初に結合されるユーザを特定する

距離行列の非対角成分で最小値を探す

表24.2を全て確認すると，最も小さい距離は $d(3,4) = 1.4$ です．

したがって，両方のデンドログラムでユーザ3と4が最初に結合されます． デンドログラム (a), (b) とも，最も底部で結合されている2点は A と C であることがわかります．

→ $\{A, C\} = \{3, 4\}$（A=3, C=4 または A=4, C=3）

Step 2：次に結合されるユーザを特定する

{3,4} クラスターと残りのユーザの距離を求める

クラスター $\{3,4\}$ と各ユーザ $i$ の距離を，最近隣法・最遠隣法それぞれで計算します：

相手 $i$最近隣（min）最遠隣（max）
ユーザ1min(3.3, 3.3) = 3.3max(3.3, 3.3) = 3.3
ユーザ2min(5.5, 5.6) = 5.5max(5.5, 5.6) = 5.6
ユーザ5min(1.6, 2.3) = 1.6max(1.6, 2.3) = 2.3
ユーザ6min(4.0, 2.7) = 2.7max(4.0, 2.7) = 4.0

最近隣法でも最遠隣法でも，$\{3,4\}$ と最も近いのはユーザ5です（それぞれ1.6と2.3）． また，$d(1,2) = 2.4$（ユーザ1と2の距離）も候補になります．

次の結合：図から $\{A,C,F\} = \{3,4,5\}$ と判断できる → F=5

デンドログラム (a), (b) とも，{A,C} に続いて F が結合されています． $\{3,4\}$ に次に結合されるのがユーザ5であることから，F = 5．

→ $\{A,C,F\} = \{3,4,5\}$

Step 3：残り B, D, E を特定する（B=6）

デンドログラムの形から D, E が {1,2} に対応することがわかる

デンドログラム (a), (b) とも，D と E が対として結合されています． 残りのユーザは1, 2, 6です．$d(1,2) = 2.4$，$d(1,6) = 5.4$，$d(2,6) = 7.5$ であり， 最も近い組は $d(1,2)=2.4$ → D, E はユーザ1と2（$\{D,E\} = \{1,2\}$），したがって B = 6．

Step 4：(a) と (b) の対応を決める

鍵は「$\{3,4,5\}$ が $\{1,2\}$ とユーザ6のどちらと先に結合するか」

$\{3,4,5\}$ クラスターと残り2者（$\{1,2\}$，ユーザ6）の距離を計算します：

比較対象最近隣（min）最遠隣（max）
$\{3,4,5\}$ vs $\{1,2\}$ min(d(3,1), d(3,2), d(4,1), d(4,2), d(5,1), d(5,2))
= min(3.3, 5.5, 3.3, 5.6, 2.5, 4.4) = 2.5		 max(3.3, 5.5, 3.3, 5.6, 2.5, 4.4) = 5.6
$\{3,4,5\}$ vs ユーザ6 min(d(3,6), d(4,6), d(5,6))
= min(4.0, 2.7, 4.6) = 2.7		 max(4.0, 2.7, 4.6) = 4.6
  • 最近隣法：2.5 < 2.7 なので $\{3,4,5\}$ は $\{1,2\}$ と先に結合 → ユーザ6（B）は最後まで孤立
  • 最遠隣法：4.6 < 5.6 なので $\{3,4,5\}$ はユーザ6（B）と先に結合 → Bが早めに{A,C,F}グループに入る
デンドログラムの形と対応させる

デンドログラム (b) では B が一番高い位置（最後）に全体に結合されています — これは最近隣法で B=6 が最後に結合される（h=2.7）構造と一致します．
デンドログラム (a) では B が $\{A,C,F\}$ グループの中に先に取り込まれています — これは最遠隣法で $\{3,4,5\}$ と6が h=4.6 で結合する構造と一致します．

結合順序のまとめ

手法順序結合される組高さ（距離）
最遠隣法 (a)①{3,4} = {A,C}1.4
②{3,4,5} = {A,C,F}（5を追加）2.3
③{1,2} = {D,E}2.4
④{3,4,5,6} = {A,C,F,B}（6を追加）4.6
⑤全体結合7.5
最近隣法 (b)①{3,4} = {A,C}1.4
②{3,4,5} = {A,C,F}（5を追加）1.6
③{1,2} = {D,E}2.4
④{1,2,3,4,5} = {A,C,F,D,E}（{1,2}が加入）2.5
⑤全体結合（6=Bが最後）2.7
結論

(a) は最遠隣法，(b) は最近隣法に対応する．

ユーザ番号の対応：A=3, B=6, C=4, D=2, E=1, F=5（A と C，D と E はそれぞれ入れ替えてもよい）

問24.1 [2] — K-means 法（K=2）のクラスター中心の計算

解答・解説
K-means 法の手順をおさらいする
  1. 初期中心の設定：K 個の初期クラスター中心 $\bar{x}_1^{(0)}, \ldots, \bar{x}_K^{(0)}$ を与える
  2. 割り当てステップ：各データ点を最も近い中心のクラスターに割り当てる
  3. 更新ステップ：各クラスターに属するデータの平均を新しい中心とする
  4. 収束判定：中心が変化しなければ終了，そうでなければ②に戻る

ユーザ6人のデータを整理する

ユーザ$x_i$（レストラン, 劇場, ビーチ）
$x_1$(7.7, 5.8, 7.7)
$x_2$(10.1, 5.9, 7.3)
$x_3$(4.9, 4.1, 7.9)
$x_4$(4.8, 5.0, 9.0)
$x_5$(6.4, 3.7, 8.0)
$x_6$(4.0, 5.2, 11.6)

初期中心：$\bar{x}_1^{(0)} = (9, 6, 7)$，$\bar{x}_2^{(0)} = (5, 5, 10)$

割り当てステップ：各ユーザと初期中心のユークリッド距離

距離の公式：$d(\bar{x}^{(0)}_k, x_i) = \sqrt{(a-p)^2+(b-q)^2+(c-r)^2}$

中心1（$\bar{x}_1^{(0)} = (9,6,7)$）との距離：

ユーザ計算式距離 $d(\bar{x}_1^{(0)}, x_i)$
$x_1$ $\sqrt{(9-7.7)^2+(6-5.8)^2+(7-7.7)^2} = \sqrt{1.69+0.04+0.49}$ $= \sqrt{2.22} \approx \mathbf{1.49}$
$x_2$ $\sqrt{(9-10.1)^2+(6-5.9)^2+(7-7.3)^2} = \sqrt{1.21+0.01+0.09}$ $= \sqrt{1.31} \approx \mathbf{1.14}$
$x_3$ $\sqrt{(9-4.9)^2+(6-4.1)^2+(7-7.9)^2} = \sqrt{16.81+3.61+0.81}$ $= \sqrt{21.23} \approx \mathbf{4.61}$
$x_4$ $\sqrt{(9-4.8)^2+(6-5.0)^2+(7-9.0)^2} = \sqrt{17.64+1.00+4.00}$ $= \sqrt{22.64} \approx \mathbf{4.76}$
$x_5$ $\sqrt{(9-6.4)^2+(6-3.7)^2+(7-8.0)^2} = \sqrt{6.76+5.29+1.00}$ $= \sqrt{13.05} \approx \mathbf{3.61}$
$x_6$ $\sqrt{(9-4.0)^2+(6-5.2)^2+(7-11.6)^2} = \sqrt{25.00+0.64+21.16}$ $= \sqrt{46.80} \approx \mathbf{6.84}$

中心2（$\bar{x}_2^{(0)} = (5,5,10)$）との距離：

ユーザ計算式距離 $d(\bar{x}_2^{(0)}, x_i)$
$x_1$ $\sqrt{(5-7.7)^2+(5-5.8)^2+(10-7.7)^2} = \sqrt{7.29+0.64+5.29}$ $= \sqrt{13.22} \approx \mathbf{3.64}$
$x_2$ $\sqrt{(5-10.1)^2+(5-5.9)^2+(10-7.3)^2} = \sqrt{26.01+0.81+7.29}$ $= \sqrt{34.11} \approx \mathbf{5.84}$
$x_3$ $\sqrt{(5-4.9)^2+(5-4.1)^2+(10-7.9)^2} = \sqrt{0.01+0.81+4.41}$ $= \sqrt{5.23} \approx \mathbf{2.29}$
$x_4$ $\sqrt{(5-4.8)^2+(5-5.0)^2+(10-9.0)^2} = \sqrt{0.04+0.00+1.00}$ $= \sqrt{1.04} \approx \mathbf{1.02}$
$x_5$ $\sqrt{(5-6.4)^2+(5-3.7)^2+(10-8.0)^2} = \sqrt{1.96+1.69+4.00}$ $= \sqrt{7.65} \approx \mathbf{2.77}$
$x_6$ $\sqrt{(5-4.0)^2+(5-5.2)^2+(10-11.6)^2} = \sqrt{1.00+0.04+2.56}$ $= \sqrt{3.60} \approx \mathbf{1.90}$

クラスターへの割り当て

各ユーザを距離が小さい方のクラスターに割り当てます：

ユーザ$d(\bar{x}_1^{(0)}, x_i)$$d(\bar{x}_2^{(0)}, x_i)$割り当て
$x_1$1.493.64→ $C_1$
$x_2$1.145.84→ $C_1$
$x_3$4.612.29→ $C_2$
$x_4$4.761.02→ $C_2$
$x_5$3.612.77→ $C_2$
$x_6$6.841.90→ $C_2$

$C_1 = \{x_1, x_2\}$，$C_2 = \{x_3, x_4, x_5, x_6\}$

更新ステップ：新しい中心を計算する

$\bar{x}_1^{(1)}$（$C_1$ の平均，$x_1$ と $x_2$ の平均）：

$$\bar{x}_1^{(1)} = \frac{1}{2}(x_1+x_2) = \frac{1}{2}\!\begin{pmatrix}7.7+10.1\\5.8+5.9\\7.7+7.3\end{pmatrix} = \frac{1}{2}\begin{pmatrix}17.8\\11.7\\15.0\end{pmatrix} = \begin{pmatrix}8.9\\5.9\\7.5\end{pmatrix}$$

$\bar{x}_2^{(1)}$（$C_2$ の平均，$x_3, x_4, x_5, x_6$ の平均）：

$$\bar{x}_2^{(1)} = \frac{1}{4}(x_3+x_4+x_5+x_6) = \frac{1}{4}\!\begin{pmatrix}4.9+4.8+6.4+4.0\\4.1+5.0+3.7+5.2\\7.9+9.0+8.0+11.6\end{pmatrix} = \frac{1}{4}\begin{pmatrix}20.1\\18.0\\36.5\end{pmatrix} = \begin{pmatrix}5.0\\4.5\\9.1\end{pmatrix}$$

収束確認：もう1回繰り返すと？

なぜ1回で収束するのか？

新しい中心 $\bar{x}_1^{(1)}=(8.9,5.9,7.5)$，$\bar{x}_2^{(1)}=(5.0,4.5,9.1)$ で再び各ユーザを割り当てると， 初期中心と似た位置にあるため割り当てが変わりません（$C_1=\{x_1,x_2\}$，$C_2=\{x_3,x_4,x_5,x_6\}$ のまま）． 割り当てが変わらなければ中心も変化しないので，アルゴリズムは1回の繰り返しで収束します．

結論：得られた2クラスターの中心座標
クラスター含まれるユーザ中心座標
$C_1$ ユーザ1, 2（レストラン評価が高い） $\bar{x}_1^{(1)} = (8.9,\ 5.9,\ 7.5)$
$C_2$ ユーザ3, 4, 5, 6（ビーチ評価が高い） $\bar{x}_2^{(1)} = (5.0,\ 4.5,\ 9.1)$
クラスターの意味を考えよう

$C_1$（ユーザ1, 2）はレストラン評価が高い（$\approx 9$）グループです． $C_2$（ユーザ3, 4, 5, 6）はビーチ評価が高い（$\approx 9$）グループです． 劇場評価は両グループとも似たような値で，差別化要因になっていません．

このようにクラスター分析の結果を「どの特徴量でグループが分かれているか」という観点で解釈することが重要です．
